Cotality - Internal Use Only

What is PagerDuty Used For?

  • Alert Routing: Verify alerts from Elastic/Uptrends route to correct services.
  • On-call Rotations: Ensure schedules are defined and maintained.
  • Incident Management: Integration into overall incident response workflow.

What is NOT Accepted?

  • ✕ Teams with no EP or Service.
  • ✕ Inactive, Temporary, or Test teams.
  • ✕ Hyphens inside names (e.g., cloud-platform). Use underscores instead.
  • ✕ Adding individual users directly to an Escalation Policy.

Object Name Validator

Test your names against the regex audit scripts.

Must follow Org-Space format.

Quick Rules:
  • Use _ inside Org/Space names.
  • Use - ONLY as the separator.
  • See Reference Table for scenarios.

Validate against the scenarios below:

Scenario Naming Format Example
Standard Team <Org>-<Space> platform_services_glb-anthos
Multi-Space Team <Org>-<Func>~Support insurance_uw_us-L2_app~Support
Cross-Org Team <Func>-Support Corp_Jira-Support

Format: ShortOrg-ShortSpace-ServiceName

Rules:
  • ShortOrg & ShortSpace must be 3 chars.
  • Special teams use bus-spt.
  • No hyphens in ServiceName (use underscores).

Example: iuu-rqe-rqecoreapp (Use 'bus-spt' for special teams)

Format: AppName-L#-Optional

Rules:
  • L# indicates tier (e.g., L1, L2).
  • Use _ instead of spaces in AppName.
  • Optional indicator for clarification.

Example: My_App-L1 or My_App-L2-Backup

Format: AppName-L#-EP

Rules:
  • L# reflects highest escalation level (Steps in EP).
  • L4-EP includes full L1->L4 chain.
  • Add Org/Space as tags, not in name.

Example: Elastic-L4-EP
Remember: Do NOT add individual users to EPs, only schedules.
